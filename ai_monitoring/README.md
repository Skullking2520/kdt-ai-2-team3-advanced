# LLM + RAG 응답 monitoring을 하기 위한 폴더

본 폴더는 llmops의 observability관련 파일들을 모아놓았습니다.

# 🚀 Langfuse 개요 및 핵심 기능 정리

Langfuse는 LLM 애플리케이션의 개발, 모니터링, 그리고 테스트 단계를 가속화하기 위해 설계된 **오픈소스 기반의 LLM 엔지니어링 플랫폼(LLMOps)**입니다. 프로토타입 단계부터 대규모 엔터프라이즈 환경까지 LLM 앱의 전체 수명 주기를 효율적으로 관리할 수 있도록 돕습니다.

---

## 📌 핵심 기능 (Core Capabilities)

### 1. LLM 옵서버빌리티 & 애플리케이션 추적 (Observability & Tracing)

- **계층적 트레이싱:** 복잡하게 중첩된 LLM 호출, 도구 사용(Tool calls), RAG 검색 단계를 트리 구조(`Trace` - `Span` - `Generation`)로 시각화합니다.
- **성능 및 비용 모니터링:** 실행 속도(Latency), 토큰 소모량, 정확한 비용 지표를 실시간 대시보드로 추적하여 병목 구간을 진단합니다.
- **대화 세션 관리:** 멀티턴 대화나 에이전트 루프 구조의 연속적인 트레이스를 하나의 `Session`으로 묶어 사용자 경험 단위로 추적할 수 있습니다.

### 2. 프롬프트 관리 시스템 (Prompt Management)

- **코드 분리 및 중앙 관리:** 코드를 재배포할 필요 없이 Langfuse UI에서 프롬프트를 작성, 수정, 버전 관리할 수 있습니다.
- **에지 캐싱(Edge-Cached):** 프롬프트 가져오기 API 성능이 독립적으로 최적화되어 프로덕션 환경의 지연 시간을 최소화합니다.

### 3. 유연한 평가 체계 (Evaluation)

- **LLM-as-a-Judge:** 다른 고성능 모델(GPT-4o 등)을 자동 평가자로 설정하여 환각률, 컨텍스트 관련성 등을 정량적으로 측정합니다.
- **Ragas 연동 및 외부 지표 바인딩:** Ragas와 같은 전문 RAG 평가 프레임워크 스크립트 결과 지표를 특정 트레이스의 `Score` 객체에 간편하게 동기화합니다.
- **휴먼 인 더 루프(Human-in-the-Loop):** UI를 통해 평가 점수를 수동으로 입력하거나, 정답 데이터셋(Golden Dataset) 구축용 아노테이션 작업을 협업할 수 있습니다.

### 4. 오픈소스 및 강력한 데이터 제어 (Self-Hosting)

- **인프라 통제권:** 외부 SaaS 환경에 데이터를 전송하지 않고 사내 독립된 인프라(Docker, Kubernetes 등)에 완전히 격리된 배포가 가능하여 민감한 프롬프트 및 개인정보(PII) 누출을 원천 차단합니다.
- **비동기 설계:** 백그라운드 배칭 및 비동기 수집(Async Ingestion) 방식을 채택하여 모니터링 도구의 도입이 주 애플리케이션의 런타임 성능에 영향을 주지 않습니다.

---

## 🛠️ 통합 및 확장성 (Integrations)

- **프레임워크:** LangChain(Callback 연동), LlamaIndex, OpenAI SDK, LiteLLM 등 대부분의 핵심 AI 생태계 및 모델 지원
- **표준 프로토콜:** OpenTelemetry(OTel) 네이티브 기반 아키텍처 지원으로 기존 인프라 옵서버빌리티 도구(Datadog 등)와의 결합이 용이

## Langfuse 화면 공유 (접속) 여부

팀원이 동일한 랭퓨즈 프로젝트에 접속하여 대시보드와 데이터를 열람하려면 다음 두가지 방법을 사용할 수 있습니다.

1. 프로젝트 멤버 초대: 랭퓨즈 ui의 project setting -> members 메뉴로 이동하여 팀원의 이메일을 초대할 수 있습니다. 팀원은 본인의 랭퓨즈 계정으로 로그인하여 화면을 함께 확인하고 협업할 수 있습니다.

2. 트레이스 공유 링크(특정 화면 공유): 대화 기록(session)이나 특정 추적(Trace)화면만 공유하고 싶다면 우측 상단의 share버튼을 눌러 공용 링크(public link)를 생성해 팀원에게 전달할 수 있습니다.
